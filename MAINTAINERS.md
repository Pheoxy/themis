# Maintainers

Themis has no named maintainer list in this scaffold yet. Repository owners are responsible for releases, security response, and policy decisions until maintainers are listed here.

Maintainer responsibilities:

- Keep the validator fail-closed by default.
- Reject changes that make AI-assisted submissions easier to hide.
- Require tests and generated documentation updates for CLI behavior changes.
- Treat false passes as higher severity than false positives.
- Keep project terminology clear enough that human developers understand what every command and option does.
